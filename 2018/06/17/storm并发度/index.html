storm并发度

发表于 2018-06-17 | 分类于 数据科学 | |
字数统计: 365 字 | 阅读时长 ≈ 1 分钟

relationship
三个部件: worker 、executor TASK

worker:

  • work中运行的是拓扑的子集
  • 一个worker 进程是从属于某个特定的拓扑的
  • 在worker进程中运行着一个或这个多个与拓扑组件相关的excutor
  • 一个运行中的拓扑就是由这些运行于Storm集群中的很多机器上的进程组成

executor:

  • 一个 executor 是由 worker进程生成的一个线程
  • 一个executor可能有一个或者多个task, 这些task为同一个组件（spout \ bolt）服务

task:

  • task是实际执行数据处理的最小单元（task不是线程）
  • 默认情况下每一个线程上运行一个task
  • 在拓扑的生命周期中每个组件的task的数量都是保持不变的。

在实际使用中，一般都是 一个executor只有一个 task.

注释：
在一个executor上运行多个ask并不能增加并行度， 对于一个executo 里的所有task, r始终只会有有一个线程，也就是说多个task只能串行的在executor上执行

这样设计的意义在于：

  • 在拓扑启动后，exrcutor的数量是可以修改的
  • 拓扑上的task个数是不可修改的
  • 默认的关系： $executors <= $tasks.

所以

  1. tasks的作用是为了增加拓展性
  2. 为了测试，如果开发机性能好的话，一个线程就可以跑很多task
